# ChainMind YOLO 接管设计说明

**目标**
将当前“功能完整但路径复杂”的桌面 AI 工具，升级为“结果导向、首分钟有价值输出”的开发者 AI 协作台。

**当前判断**
- 产品阶段：高级 MVP（能力强，体验未打穿）
- 最大短板：首次价值达成时间过长（TTFV）
- 核心风险：功能堆叠导致认知负担高，结果展示不够前置

## 设计原则

1. 结果优先：默认展示交付结果，不先展示系统复杂度。
2. 模板优先：高频场景直接开跑，不要求用户先搭工作流。
3. 复用优先：优先集成成熟高星开源能力，避免重造轮子。
4. 数据闭环：记录任务、链路、结果、评分，形成推荐与自动优化。

## 北极星指标

- TTFV（首次可见成果时间） < 3 分钟
- TTR（任务到可交付结果）持续下降
- Reuse Rate（模板复用率）持续上升
- Outcome Score（用户评分）持续提升
- Rework Rate（返工率）持续下降

## 目标架构（乐高式）

- 保留现有核心：Electron + Next + 本地执行 + 现有聊天/链路界面
- 编排思想：借鉴 LangGraph 的状态图与检查点模型
- 连接器策略：借鉴 n8n 的节点契约与插件机制
- 文档摄取：接入 MarkItDown 作为统一文档转 Markdown 管道
- 记忆升级：沿用现有 memory-system，按 mem0 思路扩展任务记忆层

## 首批落地范围（第一阶段）

1. Outcome Launcher：工作区首屏改为任务导向启动器。
2. Quick Templates：内置高频开发模板并一键执行。
3. Result-First Layout：结果面板前置，过程信息可折叠。
4. Feedback Loop：执行后快速评分并写入本地统计。

## 非目标（本阶段暂不做）

- 不做完整通用工作流平台重构
- 不做多租户企业权限系统
- 不做远程云端账户体系

## 成功标准

- 新用户无配置或最低配置下，1-3 分钟得到“可复制/可导出”的产物
- 模板任务完成后可自动生成结构化交付摘要
- 本地可看到近 7 日任务质量趋势
